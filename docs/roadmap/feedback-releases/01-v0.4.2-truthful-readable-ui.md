# v0.4.2 — Truthful and readable race engineering UI

## Shippable outcome

Apex no longer labels every decoded LMU session as a race, persistent fuel
calibration accepts only eligible local-player evidence, and the Live, Fuel and
Strategy interfaces are readable without relying on 6–9 px text. The release
does not add a strategy feature; it establishes the trustworthy input and
responsive typography foundation needed by later releases.

This release completes APX-000004 and the larger-text portion of APX-000003.
APX-000003 remains open for layout customization, and APX-000005 remains open
for the adaptive fuel plan.

## Scope boundaries

Included:

- represent an unavailable session kind explicitly;
- prevent replay, AI and remote control from becoming durable personal fuel
  calibration;
- introduce semantic typography tokens and responsive text containers;
- update Live, Fuel, Strategy and the shared shell/card primitives;
- prove English/German layouts at 100–200% scaling.

Excluded:

- decoding a speculative session-type offset;
- changing the packed bridge contract;
- Virtual Energy or tyre strategy;
- global user-configurable UI scaling;
- dashboard card reordering or resizing;
- broad visual redesign unrelated to readability.

## Truth and behavior contracts

### Session kind

- Add `unknown` to `SessionKind` in `src/core/types.ts`.
- Map the current bridge session kind to `unknown` in
  `src/core/desktop-adapter.ts` instead of hard-coding `race`.
- Audit every consumer of `SessionKind`. Unknown must render as “Session” /
  “Session”, not as race, practice or qualifying.
- Do not infer kind from opponent count, phase, session duration, maximum laps,
  track, server state or the presence of vehicle telemetry.
- Add a documented capability boundary beside the adapter contract.

### Fuel evidence eligibility

- Split current-frame observation from persistent calibration eligibility.
  Current fuel, tank and session progress may continue updating while AI is in
  control, but the completed lap must not be appended to the user's durable
  fuel/lap-time samples.
- Durable eligibility requires:
  - source `lmu-shared-memory`;
  - vehicle telemetry available;
  - control owner `local-player` for the complete sampled lap;
  - no pit lane contact or detected refuel;
  - existing finite/range checks.
- A lap becomes ineligible if control ownership changes at any point during it.
- Replay remains observational and must neither read nor write durable fuel
  profiles.
- Keep a bounded exclusion reason in tracker state for tests and UI diagnostics;
  do not store driver identities.

## Typography implementation

### 1. Create semantic tokens

Define relative units in the shared styles rather than multiplying existing
selectors:

- metadata/caption;
- field label;
- body/support copy;
- card title;
- primary numeric value;
- tabular/monospace data.

Initial values should put meaningful support text around 11–12 px equivalent,
ordinary content around 13–14 px, and card titles around 15–16 px at 100%.
Exact values are accepted through visual testing, not by treating these ranges
as an accessibility standard.

### 2. Remove rigid text containers

- Replace fixed heights with `min-height` where text can wrap.
- Use `rem`, `em`, `minmax()` and content-sized rows.
- Give grid children `min-width: 0` and define intentional wrapping or
  accessible disclosure for truncation.
- Keep primary telemetry numerals tabular and stable as values change.
- Do not reduce font sizes again inside narrow media queries to preserve the old
  density.

### 3. Apply the first pass

Update, in order:

1. shared badges, fields, buttons, cards, navigation and status text;
2. measured Live cards and their unavailable/session-only states;
3. Fuel Calculator automatic/manual modes and result cards;
4. Strategy inputs, candidates, unsupported factors and provenance;
5. Settings lifetime summary where it shares the same primitives.

Tiny decorative SVG annotations may remain smaller only when the same
information is available as readable text or an accessible name. No actionable
control may depend on tiny text.

## Infrastructure and persistence

- No SQLite migration.
- No new preload IPC.
- No bridge protocol change.
- Existing localStorage fuel-profile keys remain unchanged.
- A new exclusion-reason field, if exposed outside the tracker, stays in memory
  and contains an enum rather than raw telemetry or identity data.

## Focused automated tests

Add or update:

- `src/core/desktop-adapter.test.ts`
  - current raw frames produce `session.kind === 'unknown'`;
  - no consumer fallback silently changes it to `race`.
- `src/core/fuel-tracker.test.ts`
  - one entirely local clean lap is accepted;
  - AI, remote and replay laps are rejected;
  - a mid-lap local→AI→local transition rejects the whole lap;
  - current fuel still updates while calibration is withheld;
  - pit/refuel/session-only behavior remains unchanged.
- `src/fuel-profile.test.ts`
  - ineligible evidence never reads, replaces or writes a stored profile;
  - an eligible live profile still survives a new session.
- renderer tests for Live, Fuel and Strategy at long German strings and
  unavailable session kind.
- a static style regression that rejects newly introduced rendered text below
  the agreed token floor outside an explicit allowlist.

The mandatory pre-push hook must additionally produce and validate the Windows
installer, portable ZIP, updater metadata and checksum file. Do not treat the
Linux build or a source-mode Electron screenshot as packaged-Windows proof.

## Acceptance criteria

- No official live frame is labelled practice, qualifying or race without an
  authoritative decoded value.
- AI, remote, replay and mixed-control laps add no durable fuel samples.
- Live current fuel remains available when calibration is withheld.
- All affected meaningful text uses the semantic scale rather than local
  6–9 px literals.
- Live, Fuel and Strategy remain usable at 200% in both languages.
- No database, bridge payload or updater contract changes.
- Existing tests and the complete repository gates pass.

2. Open Live without LMU, with scoring-only data, and with the player car active.
3. Run one local-player clean lap and one AI-controlled lap; only the local lap

4. Open Live, Fuel and Strategy in English and German at 100%, 150% and 200%.
5. Restart twice and confirm data and layout remain usable.
6. Check the in-app What’s New entry offline.

shipped while layout customization remains planned; keep it `in_progress`.

## Rollback and definition of done

Rollback is code-only because this release has no migration. Downgrading must
not corrupt existing profiles or SQLite stores. The milestone is done only when
the published installer passes the post-release checks, APX-000004 has a
verification-backed resolution, APX-000003 records its shipped partial scope,
and APX-000005 accurately describes its remaining scope.
