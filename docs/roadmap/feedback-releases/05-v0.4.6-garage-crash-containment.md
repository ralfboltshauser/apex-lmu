# v0.4.6 — Garage crash containment

## Shippable outcome

Opening Garage cannot take down the React view when the Electron IPC response
is missing, rejected or structurally incompatible. A valid local lifetime
ledger still renders the exact same measured cars, tracks and integer-millimetre
totals. An incompatible response produces the existing recovery state, records
a local diagnostic and leaves `apex.sqlite3` unchanged.

The incident report arrived outside the in-app feedback thread. Its exact
private lifetime database is not available on this host, so the affected PC is
the final compatibility test. The private raw recording cannot substitute for
that database: by design `.apexrec` contains LMU snapshots, not lifetime SQLite.

## Implementation

1. Validate the complete `ApexGarageStats` response before setting React state:
   lifecycle status, schema/catalog versions, totals, models, tracks, counts,
   dates and finite distances.
2. Reject an incompatible response atomically, clear pending history, report
   only the contract error to local diagnostics and render the existing
   Data & storage recovery path.
3. Wrap Garage in a view-level error boundary so a later render failure records
   a local diagnostic and keeps the recovery controls usable.
4. Extend the real Windows desktop replay E2E to seed one measured car and
   track in schema-v1 SQLite before launch, open Garage through preload IPC,
   verify the expanded model/track, and preserve its exact total through raw
   replay, explicit import and full app restart.
5. Do not migrate, repair, replace or upload the lifetime database.

## Verification and release proof

- focused React regression: a model with an incompatible `tracks` field shows
  recovery and produces no partial car list;
- lifetime SQLite and catalog suites still reconcile exact totals;
- strict private replay confirms scoring-only and LMU-garage frames retain car
  identity, without printing private identities;
- render the seeded German Garage in Electron, capture a screenshot and inspect
  it for the complete model, track, totals and absence of renderer errors;
- run the full repository gates, Windows bridge cross-compile and high-severity
  audit;
- push through the release hook, wait for Linux verification, the real Windows
  bridge/E2E job and Vercel, then publish the installer, ZIP, updater metadata
  and checksums;
- verify the public tag, assets, SHA-256 values, website filenames and installed
  packaged Garage before asking the affected user to open Garage once.

## Rollback and definition of done

Rollback is code-only because schema v1 and all ledger rows remain unchanged.
The release is done when public v0.4.6 passes the seeded source and packaged
Garage path and the affected PC confirms that opening Garage no longer crashes.
